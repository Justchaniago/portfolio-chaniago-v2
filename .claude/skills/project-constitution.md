PROJECT CONSTITUTION

This document overrides all other instructions.

Primary Goal

Maintain and improve the existing product.

Do not reinvent it.

Do not redesign it.

Do not reinterpret it.

The existing implementation is the source of truth.

⸻

Success Criteria

A successful task:

* solves the requested problem
* preserves existing behavior
* preserves existing visual identity
* introduces minimal regression risk

A successful task is NOT:

* creating a new design
* introducing new patterns
* rewriting architecture unnecessarily

⸻

Scope Discipline

Only modify what was requested.

Do not modify unrelated sections.

Do not refactor unrelated code.

Do not rename files unless requested.

Do not move components unless necessary.

⸻

Modification Hierarchy

Always prefer:

1. Styling adjustment
2. Layout adjustment
3. Component adjustment
4. Local refactor
5. Structural refactor
6. Architecture change

Never jump directly to 5 or 6.

⸻

Existing Code Assumption

Assume the existing code exists for a reason.

Before removing anything:

Ask:

* Why was this implemented?
* What depends on it?
* What breaks if removed?

⸻

Portfolio Identity

This project is:

* editorial
* premium
* monochrome
* cinematic
* motion-driven

This project is NOT:

* SaaS dashboard
* startup landing page
* dribbble concept
* trendy redesign experiment

Do not introduce design language outside this identity.
